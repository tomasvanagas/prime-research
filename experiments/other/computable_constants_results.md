# Computable Real Number Approaches: Results

**Script:** computable_constants.py

## What Was Tested
Six approaches using computable real constants to encode primes: theta*n*ln(n) model, binary encoding constants, BBP-type digit extraction, iterated function systems, dynamical systems producing primes, and automatic sequences / finite automata for primes mod m.

## Key Findings
- theta(n) = p(n)/(n*ln(n)) converges to 1 but is NOT constant; deviations are unpredictable without knowing p(n)
- Binary encoding constants (e.g., Copeland-Erdos) require knowing primes to compute their digits
- BBP-type extraction is specific to certain constants (pi, log2); no analogous formula exists for prime-encoding constants
- No iterated function system or dynamical system was found that generates primes in polylog time
- Primes mod m are not generated by any finite automaton (not an automatic sequence in any base)

## Verdict
**CLOSED** -- Failure Mode: C (Circularity) / I (Information Loss)

## One-Line Summary
No computable real constant provides a polylog shortcut to individual prime values; encoding constants require primes to compute.
